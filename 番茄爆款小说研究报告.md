## 摘要（先读这一页）

番茄小说是字节跳动旗下免费阅读 App（2019.11 上线），用**抖音同源的推荐算法**做内容分发，本质是**"用故事留住用户时长，再把时长卖给广告主"**的流量生意。对创作者（含 AI 生成系统）的全部含义可压缩成一句话：

> **你的第一读者不是人，是算法；算法只认三件事——吸量（点击率）、完读率、追读率，三者乘积即"总留存"，总留存决定流量池升降。**

由此推导出爆款的底层公式：

```
爆款 = 吸量(书名+封面+简介) × 追读率(钩子密度) × 完读率(节奏稳定) × 三日留存(情绪价值持续)
     ──────────────────────────────────────────────────────────────────
                     在"对的赛道"上，用"对的人群"放大
```

**七个最高优先级结论（按 ROI 排序）：**

1. **3 秒定生死 → 黄金三章已进化到"黄金三句"。** 开篇第一段必须是"正在发生的危机"，不是背景铺垫。
2. **追读率是生命线，经验生死线 ≈ 50%**；完读率决定赚多少钱，追读率决定生死。
3. **8 万字进 7 天验证期（约 5000 测试流量），10 万字首秀（14 天）**。10 万字完读率行业均值 16–17%，>17% 优秀，巅峰作 >23%；新书池三日留存 ≥50% 是爆款门槛。
4. **下沉市场是基本盘**：番茄 58.6% 用户来自三线及以下城市，一线仅 7.5%；用户与抖音重合 72%。语言要"大白话"，金手指要"简单明了"。
5. **节奏即奖励频率**：追读率 top10% 作品平均每 **1.8 章**一个情绪高峰，bottom10% 是 4.7 章。每章≥1 小爽点，3–5 章 1 中爽点，每卷 1 大爽点。
6. **去类型化是 2024–2026 最大风向**：男女频融合、老白小白通吃的"新网文"才能登巅峰榜（《十日终焉》《斩神》范式）。
7. **短剧反哺写作**：番茄 IP → 红果短剧闭环。强人设、强情感、强反转的"可短剧化"结构=额外变现+流量加权。

# 一、番茄小说平台规则分析

## 1.1 推荐机制：一台"没有感情的总留存计算器"

番茄算法与抖音/小红书同源，核心不是"判断书好不好"，而是**预测用户会不会停留**。可用一个公式概括作者社群与产品分析的共识：

```
流量池PK实力 = 总留存 = 投放流量 × 封面点击率(吸量) × 阅读留存(完读总和)
```

- **投放流量**：当前流量池一次性给的展示位数量（同池作品相同）。
- **封面点击率 / 吸量**：书名+封面+简介把人"骗"进正文的能力。
- **阅读留存**：进来的人累计读完了多少章（对应短视频的"停留秒数总和"）。

算法据此把作品划入**晋升档 / 不变档 / 下降档**，决定下一个周期的流量池。

### 新书冷启动全流程（作者社群经验值，非官方）

| 阶段 | 字数节点 | 周期 | 核心指标 | 经验阈值 |
|---|---|---|---|---|
| **单机期** | 0 – 8 万字 | 不定 | 稳定日更、签约 | 日更 4k–6k 不断更 |
| **验证期/小流量A/B** | 约 8 万字触发 | 7 天 | 吸量分级（D→SSS 共 7 档）+ 留存训练人群画像 | 约 5000 测试流量；点击 300+ 评 A，500+ 评 SS |
| **首秀期** | 约 10 万字 | 14 天 | **10 万字完读率 + 追更比** | 完读率均值 16–17%，>17% 优秀，巅峰 >23%；追更比理想 2:1 |
| **书测/放量节点** | 20 万 / 50 万 / 100 万字 | 滚动 | 总量留存 PK | 每过节点再测一次，可二次起飞 |
| **完结推** | 完结 | 随机 | — | 非所有作品 |

### 流量池分层（社群推测）

```
验证流量池(10–13w字)  → 奖励/争夺首秀流量大小
新书流量池(13–30w字)  → "养蛊"主战场：13w后流量腰斩多发生在此（被同池新书PK掉）
老书流量池(30w–完结)  → 很多书在30w后起飞：对手断更/你的总量留存反超
```

**关键洞察：番茄不把新书和老书放一个池子**，所以"30 万字后起飞"现象普遍——竞争者断更，或你的累计完读总和反超。

## 1.2 各权重指标拆解（按重要度排序）

| 权重 | 是什么 | 真实地位 | 对生成系统的指令 |
|---|---|---|---|
| **追读率（追更比）** | 在读人数 vs 追到最新章人数（"在读"=过去 14 天读过的人） | **生命线**。追读稳=有市场价值=不限流；理想 2:1，经验生死线≈50% 追到最新 | 每章末必须留钩子；3 章内不得有"平路"超载 |
| **完读率** | 点击数 vs 读完某字数节点人数 | **决定赚多少钱**，不决定生死。完读高→评分高→曝光大 | 章内节奏不能塌；前期完读曲线下滑要"平稳" |
| **三日留存** | 新增读者中第 3 天仍在读的比例 | **评分核心**：1% 留存≈7.7–7.8 分，50%+≈9.9–10 分；爆款门槛 ≥50% | 第 1–3 章必须建立"我明天还想看"的钩子链 |
| **吸量/点击率** | 书名+封面+简介把曝光转化为点击 | 漏斗第一层，烂书名可干掉 90% 机会 | 标题=一句话爽点；简介三段式 80–120 字 |
| **阅读时长** | 单用户停留总时长 | 直接=平台广告收益=单价 | 章节足够长（建议 2.5k–3k）、连贯不出戏 |
| **评分** | 50 个有效书评（读满 30 分钟）开分；规模大后由留存决定 | 影响曝光次数；与读者打分关系小（"1000 人打满分也没用"） | 别刷分，刷留存 |
| **互动（段评/章评）** | 每万字互动密度（≥500 条/万字为 S 级）、书友圈日活 | 巅峰榜维度之一；可截图金句驱动段评爆发 | 制造"可截图传播"的金句钩子 |
| **催更** | 看广告催更=作者收益+热度信号 | 辅助热度信号 | 章末悬念 → 诱发催更行为 |

## 1.3 两个被严重低估的"非算法"流量入口

1. **书荒广场（人工社区）长期占总流量 30–40%！** 超三分之一阅读行为发生在社区（话题+书单）。它兼具"读者找书 / 推书变现 / 作者自救"三功能。→ **可被书友主动安利、进书单的作品**额外吃一大块自然流量。设计含义：要有"一句话能被推荐出去的记忆点"。
2. **巅峰榜（人机协同）**：2023.10 推出，每月 TOP30，年度 TOP10。6 个维度=**作品人气、内容质量、口碑评分、读者互动、传播价值、IP 潜力**。核心硬指标（媒体披露）：首秀期 10 万字完读率 >23%、新书池三日留存 ≥50%、差评率 <10%、每万字互动 ≥500 条（S 级）。**编辑只调权重，不直接选**。

## 1.4 平台属性与生态（影响策略的硬背景）

- 模式：**免费阅读 + 广告分成**；作者稿费=平台分的"广告费"，与字数、完读率、阅读时长、读者数挂钩，**不透明、按 14 天周期波动**。
- 规模：MAU 约 1.9 亿（2023.12，QuestMobile），2024 年称月活 2 亿+；签约作者 2024 底 **60–70 万**，lv5+ 约 2900 人，金番 39 人。
- 收入分布极陡峭：过去 12 个月 **9374 位作者年入超 3 万，3228 位超 10 万**——即头部固化、中尾部"养蛊"。
- 全勤奖门槛 2024 底从 200 元提到 500 元分成收益，新人稳定收益门槛抬高。
- AI 态度激进：番茄提供改写/扩写/续写/起名/灵感等 AIGC，与作者等级绑定（等级越高每日次数越多）。2026 年初平台同时面临"AI 生成内容泛滥导致部分作者流量下滑"的争议——**意味着纯模板 AI 文同质化会被算法和读者双重淘汰，差异化是 AI 生成的生死线**。

# 三、爆款小说底层公式

## 3.1 核心公式

```
爆款 = 强冲突 × 高频爽点 × 持续期待感 × 高留存节奏
```

展开为可执行版（生成系统用）：

```
可持续爽感 = (资源稀缺率 × 预期阈值) / [(实力通胀率 + 副作用系数) + 跨期折现]
剧情崩解   = 1 − 可持续爽感
```

- **资源稀缺率**：金手指/奖励不能无限白给（边际效用递减）。
- **预期阈值**：靠"惊喜/意外/不可预期的收获"触发多巴胺（reward prediction error）——**可预期的、规律的奖励不再分泌多巴胺**。
- **副作用系数**：力量要有代价（防止读者麻木）。
- **跨期折现**：短期爽点 + 中期伏笔（每 ~20 章回收一次）+ 长期主线，把"读者耐心折现率"控制在 0.6–0.8 的最佳成瘾区间。

## 3.2 四个"停不下来"的心理学机制

| 现象 | 机制 | 写作落点 |
|---|---|---|
| **为什么停不下来** | 多巴胺预期误差：大脑奖励的是"意外收获"。章末钩子制造"预期未闭合" | 每章末留未解信息/反转/情绪炸弹 |
| **为什么熬夜追更** | 蔡格尼克效应（未完成事件占用记忆）+ 即时满足 | 钩子吊在半空切章，绝不"圆满收尾" |
| **为什么愿意催更** | 损失厌恶 + 参与感（看广告催更=低成本投入产生掌控感） | 制造"明天就揭晓"的强悬念 |
| **为什么看上百万字** | 阶梯式延迟满足：短中长三层伏笔交替回收，成瘾不过量 | 主线缓推、支线快爽，伏笔节律性回收 |

## 3.3 六大情绪补偿（爽感的"精神来源"）

网文爽感来自**现实痛感的想象性克服**。六个最高频补偿：

1. **多巴胺循环**：高频小奖励（升级、打脸、捡漏）。
2. **成就感**：从无名到功成，可量化的"变强"。
3. **逆袭感**：阶层/地位反转（赘婿、废柴、寒门）——下沉用户最强共鸣。
4. **代入感**：主角=理想化自我（"化身斩神""我也能怼天怼地"）。
5. **情绪补偿**：现实受压→虚拟世界"打破常规又绝对安全的脱轨体验"。
6. **社会认同**：金句可截图传播（"大夏境内，神明禁行""白天当儿子晚上当老公"），形成圈层身份与二创。

# 四、番茄爆款框架拆解（按生命周期）

> 以下是**结构骨架**（从 20+ 巅峰榜/短剧爆款逆向归纳），不复制任何具体作品文本。

## 4.1 开篇·前 3 章（黄金三章 → 黄金三句）

**2026 共识：3 秒定生死，决策发生在第 3 段。** 三章结构：

- **第 1 章（钩子章）**：开篇 3 句内甩出"正在发生的危机"+ 主角核心反差 + 金手指的影子 + 章末金句钩子。**禁止**：大段景物/设定/云山雾罩对话/童年回忆。
- **第 2 章（爽点验证章）**：兑现第一次小爽点（小打脸/小逆转），证明"这书会爽"，确认人群画像。
- **第 3 章（绑定章）**：抛出第一条长线期待（主线目标/三年之约式承诺），让读者"加书架"。

开篇三要素的**递进顺序**（不可乱）：冲突前置（让人停下）→ 人设反差（让人记住）→ 金句钩子（让人想看下一章）。

## 4.2 黄金十章·前 10 万字（首秀决胜）

目标：把 10 万字完读率拉到 >17%（冲巅峰则 >23%），三日留存冲 ≥50%。

```
Ch1-3   钩子链：危机→小爽→长线承诺（决定追读率起点）
Ch4-6   金手指完整展示 + 第一个完整"打脸闭环"（铺垫压制→反转碾压→围观震惊）
Ch7-8   引入第一个像样的对手/势力，把冲突升一级
Ch9-10  小高潮 + 抛出"更大的世界/更大的敌人"——制造首秀末的强钩子，骗过首秀放量
```

## 4.3 第一高潮·30 章内

第一卷 Boss 战/身份揭晓/多伏笔首次集中回收，给"想发朋友圈"的大爽点。30 章是"读者从尝鲜转为忠实"的临界点。

## 4.4 中期展开·100 章结构（卷制）

以"卷"为单位（每卷 20–40 章）：每卷一个反派、一个核心冲突、一次实力跃迁、一次大爽点；卷末换地图/换势力/升维度，防止"平路"超载。每 ~20 章回收一次中期伏笔（折现率控制法）。

## 4.5 长线运营·300 章 与 千万字追读逻辑

- **三线伏笔嵌套**：短期（本章打脸）/中期（20 章后解锁新线）/长期（300 章终极对决）。参考《大奉打更人》税银案结构（短中长三层），可使追更率提升约 41%。
- **阈值管理**：用资源稀缺、副作用、新地图/新规则不断"换刺激"，对抗读者麻木。
- **断更=自杀**：算法对断更直接降权；千万字靠的是"稳定日更 + 节律化爽点"工业化产能，不是灵感。

# 七、爆款节奏控制系统

## 7.1 单章节奏（每章必备三件套）

每章 ≥1 个**小爽点** + 1 个**冲突/推进** + 1 个**章末钩子**。章长建议 2.5k–3k 字（兼顾阅读时长与完读率）。

**通用单章模板「起承转爽」**：
```
起(章首~500字)  回顾上章 + 引出本章冲突
承(中段~800字)  冲突升级、压力增大
转(~300字)      转折/敌人露底牌/绝望转希望
爽(~400字)      主角反击/真相揭晓/意外收获 + 章末钩子
```

**章末钩子四型**：悬念式 / 反转式 / 情绪炸弹式 / 信息投放式。**钩子必须提前在大纲里规划，不是写完再想结尾。**

## 7.2 多周期循环节奏框架

| 循环 | 频率 | 投放内容 | 数据对应 |
|---|---|---|---|
| **3 章循环** | 每 3 章 | 一个中爽点（完整打脸/关系突破/阶位跃升）；禁止"平路"超 3 章 | 追读率 |
| **10 章循环** | 每 10 章 | 一个小高潮 + 一条新长线钩子（首秀决胜单位） | 完读率/三日留存 |
| **30 章循环** | 每 30 章 | 一卷 Boss 战/身份揭晓/多伏笔回收（大爽点） | 留存/口碑评分 |
| **100 章循环** | 每 ~100 章 | 换地图/升维度/世界观扩张 + 中期伏笔总回收 | 长线追读 |

**量化基准**：追读率 top10% 作品平均 **1.8 章/情绪高峰**；bottom10% 为 4.7 章/高峰。**生成系统应把"情绪高峰间隔"作为硬约束（≤2 章）。**

## 7.3 节奏失败的自检信号

- 追读率下滑 → 回看最近 3–5 章：钩子是否塌？是否连续平路？
- 完读率低 → 章内节奏问题：是否前段拖沓、信息密度不足？
- AI 生成最大病灶：**节奏太均匀**（每段等长、密度一致）。必须在蓝图中显式标注"本段加速/本段慢下来"，用短句加速、长句减速。

# 八、AI 创作专用框架《番茄爆款小说 AI 生成框架》

> 本节为**可直接喂给生成引擎**的结构化规格。下方 YAML/JSON 风格块可被 `prompt.md` / `config.yaml` / craft 规则直接吸收。

## 8.1 题材选择公式

```
选题得分 = 赛道需求度(搜索/榜单热度)
         × 差异化空间(细分蓝海，避红海)
         × 短剧化潜力(强人设+强反转)
         × 团队能力匹配
策略：放弃主流大类正面硬刚 → 在"有人看但没人写好"的细分组合切入
      例：都市×直播×神医×宠妻 / 末世×囤货×基建×无敌 / 系统×心理咨询 / 重生×民间手艺
```

**2026 赛道地图**：
- **上升/蓝海**：游戏现实主义、无限流+规则怪谈、末世基建囤货、现实题材（年代文/基层视角）、银发题材、怪诞国风、"去女频化"大女主、男频甜宠。
- **红海（慎入，需强差异化）**：传统赘婿战神、霸总甜宠、纯系统升级、传统玄幻打怪。
- **衰退**：无差异化的种马后宫、纯套路歪嘴龙王式短剧文（边际递减）。

## 8.2 爆款书名公式

```
书名 = 一句话剧透最大爽点/冲突，大白话、强画面、可短剧化
模板：
  [反差身份]+[逆天行为]      《老娘心理学博士，进宫嘎嘎乱杀》
  [关系]+[冲突动作]          《唐总，离婚后请自重》
  [开局]+[金手指]            《开局地摊卖大力》/《开局长生万古》
  [处境]+[真实感钩子]        《被骗缅北，真实遭遇》
  [设定]+[态度]              《癫，都癫，癫点好啊》
规则：男频突出"系统/无敌/重生/战神"；女频突出"甜宠/闪婚/马甲/重生/团宠"
禁忌：文艺腔、抽象、看不懂的双关
```

## 8.3 爆款简介公式（三段式，番茄 80–120 字）

```
第1段 主角身份 + 开局困境（一句话给标签，不铺垫）
第2段 核心反差/独家能力（最值钱的一句，卖设定，越具体越好）
第3段 情绪承诺 + 钩子收尾（留未解谜团，禁剧透结局）
风格：大白话、强冲突、爽点前置、禁形容词大杂烩、禁全程"TA"指代
```

## 8.4 爆款开篇公式（黄金三句）

```
句1：正在发生的危机（动作/对话/具体，禁背景）
句2：主角的核心反差（弱外表/强承诺，或反常行为）
句3：章末金句钩子（情绪爆发 / 认知颠覆 / 后果预告，独立成段、可截图）
约束：第1章出主角；人名≤5个；禁大段设定/景物/童年
```

## 8.5 结构蓝图

**前 10 万字（首秀决胜，目标完读率>17%、三日留存≥50%）**

## 8.8 生成系统硬约束（写进 config / 评审器）

```yaml
hard_constraints:
  opening_first_sentence: "危机正在发生（动作/对话/具体），禁背景铺垫"
  emotion_peak_interval_max_chapters: 2      # 情绪高峰间隔上限
  per_chapter_hook: required                 # 每章末钩子
  flat_chapters_max_consecutive: 3           # 连续"平路"上限

  golden_first_10w_completion_target: 0.17   # 首秀完读率目标

  golden_finger_complexity: low              # 金手指越简单越好
  reward_predictability: low                 # 靠意外触发多巴胺，禁规律白给
  resource_scarcity: enforced                # 金手指/奖励须有代价与稀缺
  differentiation: enforced                  # 反同质化（AI 文淘汰线）
  voice_register: 下沉友好、低阅读门槛、短句驱动、对话优先
  shortdrama_ready: true                     # 强人设+强反转，可短剧化
anti_patterns:                               # 毒点/雷区（强惩罚）
  - 开篇大段景物/设定/世界观
  - 慢热铺垫超过3章
  - 章末圆满收尾（无钩子）
  - 节奏均匀（每段等长、密度一致）→AI 通病
  - 复杂金手指/绕口设定
  - 形容词堆砌/文艺腔/全程"TA"指代
  - 逻辑/数据错误、错别字
  - 力量无代价导致爽感通胀
```

## 8.9 与本仓库引擎的对接建议

- **`planning.py` 策略库**：把 8.7 冲突库 + 8.6 爽点库映射为候选计划的"required_constraints"；把 7.2 多周期循环作为卷级 `volume_plan` 锚点。
- **`quality.py` / `review.py`**：把 8.8 `hard_constraints` 做成确定性惩罚项（情绪高峰间隔、章末钩子、连续平路、章长、金手指复杂度），与 `style_health` 同级，**减分而非劝退式建议**——对抗 LLM 自评分膨胀。
